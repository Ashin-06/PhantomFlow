Technical & Architectural Guide

Welcome to the complete deployment, architecture, and dashboard operations manual for the PhantomFlow encrypted traffic threat detection framework. This guide details the backend pipelines, machine learning ensembles, rule engines, active response models, and SOC workflows.

1. Data Acquisition & Processing Pipeline

PhantomFlow runs an asynchronous, highly efficient pipeline from raw packet capture to dashboard telemetry. The diagram below details the data flow through all systems:

📡 Acquisition & Ingestion

The **Packet Sniffer** captures network frames in real-time. To optimize performance, the sniffer applies defensive filters, discarding loopback packets (IP `127.0.0.1` and `::1`) and internal service ports (Redis, Kafka, and PostgreSQL sockets) so that only host-to-host network traffic enters the pipeline.

Flow structures are serialized as JSON payloads and written to the network-flows **Kafka Message Broker** queue, which decouples network sniffing from memory-intensive model evaluation.

🤖 Threat Classification

The **Pipeline Orchestrator** pulls flows from Kafka, extracts statistical features, and routes them through parallel classification engines:

  • **ML Ensemble**: LSTM networks detect C2 beacon timing anomalies, while Isolation Forests compute exfiltration thresholds.
  • **Rule Engines**: Memory-efficient Redis HyperLogLog structures track source IPs to flag horizontal scans, vertical scans, brute-force attempts, and multi-signal ransomware signatures.

Hunt Tab (Historical Threat Search)

The Hunt tab allows analysts to run deep-dive queries across all historical alerts saved in the PostgreSQL database. Unlike the live feed, the Hunt tab includes already reviewed alerts and displays status badges representing their verdict.

Response Queue Tab (Active Mitigation)

The Response Queue implements human-in-the-loop validation for active response commands. Whenever an analyst initiates a mitigation action, a record is added to this queue where senior engineers approve execution.

  • 🔒 IP Block: Blocks external destination C2 IPs on perimeter Palo Alto Panorama firewalls.
  • 🌀 Sinkhole Domain: Forces DNS sinkhole policies on Cloudflare Gateway DNS servers.
  • 🛡️ Isolate Host: Quarantines compromised internal computers using EDR APIs (CrowdStrike Falcon).

Suppression Rules Tab (Noise Mitigation)

Suppression rules filter out repeating, verified benign alerts (e.g. internal backup server transfers generating exfiltration alerts) to keep the SOC feed clean and focused. Match rules support CIDR ranges, destination ports, and regex patterns.

4. Technical Setup & Maintenance

To run the developer integration test suite or reset state telemetry:

# Initialize database schema from scratch
python scratch/init_db.py

# Wipe PostgreSQL logs & flush Redis counters
python clear_db.py

# Launch all Python microservices (FastAPI uvicorn, Sniffer, Orchestrator)
python start_all.py

Alternatively, Windows systems can double-click the run_phantomflow.bat script to run these steps automatically.
